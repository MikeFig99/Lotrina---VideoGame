XfrogPlants Libraries, Version 2.0 ( 01 March 2004 )


Whats New in Version 2.0 ?

Most models and textures have been completely rebuilt, for higher quality and more accurate representation of the plants. Many new formats are now available to accompany the Xfr (Xfrog) format. These currently include: 3ds max, Maya, Lightwave, C4D, Bryce, Vue d' Esprit, OBJ, 3DS. CDROMS include one of the above formats, DVDs include all the formats. All Models are now also rendered and supplied to you both as Perspective Renders and as High Quality Billboards (Sprites). Both are TIFF with Alpha - you can use them with their Alpha cutouts as texture maps onto single polys (or crossed polys) as background elements, or simply drag and drop them into landscape images. All PDFs have been completely re-designed and now show all Models.

 

Folder Contents and Usage

 

/Models

- 60 Files of your favorite 3D format, (if you have purchased DVDs - all available formats)
- 60 Xfrog (.xfr, requires Xfrog 3.5 / Xfrog 4.x, or XfrogTune)
- 40+ Textures (.tif, with Premultiplied Alpha)

/Billboards

- 60 Billboard Images (.tif, with Premultiplied Alpha)
(Open in Photoshop and you will see they are on transparent background and can be dropped into any image - or use these as texture maps onto single polygons in Maya as useful background elements - or use it on 2 polys ( X ). i.e. the billboards are centered.)

/Perspectives

- 60 Perspective Renders (.tif, with Object Buffer Channel)
(Open in Photoshop -> Select -> Load selection ->Select the Object Buffer-> Drag the model into the background you want.)

/Documentation

- Multi-language PDFs (Print Quality)

/Plugins
  - Several Import Plug-Ins can be found inside this folder, you can use them with the most popular 3D applications.
 
 
Texture Path

Please be sure, before you render anything, to set your 3d application texture path to include your CDROM / DVD drive, so that the models can render with their textures, or copy all the files to a directory that is on your texture path. If you use Cinema4D, please be sure that Quicktime is installed, as Cinema4D uses Quicktime libraries to decompress LZW textures.
